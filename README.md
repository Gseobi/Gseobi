## About Me

중단되지 않는 시스템과 **운영 안정성**을 중심에 두고 개발하는  
**Java Backend Developer**입니다.

이전에는 자동화 설비 분야에서 프로그램 개발 경험을 쌓았으며,
그 과정에서 **운영 안정성의 중요성**, **장애 발생 시 신속한 복구**, **상태 무결성 보장**과 같은 실무 관점을 익혔습니다.
현재는 이러한 경험을 바탕으로 사내 백엔드 전임 담당자로서 서비스 개발 및 운영 전반을 맡고 있습니다.

실제 서비스 운영 환경에서 다음과 같은 백엔드 시스템을 직접 설계·개발·운영해왔습니다.

- **Java / Spring 기반 Backend API** 설계 · 개발 · 운영
- 다수 PG사를 연동하는 **결제 분기 Gateway 구조** 설계 · 개발 · 운영
- Socket, HTTP/HTTPS, SMTP, FCM Push 기반 **장기 실행 Daemon 아키텍처** 설계 · 개발 · 운영
- 이중화 환경에서의 **Scheduler 중복 실행 방지 구조** 설계 · 개발 · 운영
- **Deferred Deep Link** 백엔드 프로세스 및 아키텍처 전담 설계
- 외부 업체 / Open API / REST / Socket 기반 **대외 시스템 연동 경험**
- 외국인 대상 App 서비스 환경에서 **외부 연동 중심 API 운영 경험**
- 장시간 처리 작업에 대해 **비동기 Thread Pool 기반 처리 구조 적용 경험**

기능 구현 자체보다,  
**실제 운영 환경에서 장애를 줄이고 안정적으로 유지되는 구조를 만드는 것**에 더 큰 가치를 두고 있습니다.

## ✅ Upload Complete

#### provider-integration-gateway
- 다수 PG사를 분기 처리하는 결제 요청 게이트웨이
- Strategy Pattern 기반 구조
- 운영형 결제 분기 구조를 포트폴리오 형태로 재구성

#### java-socket-daemon-springboot
- Spring Boot 기반 장기 실행 Socket Daemon
- 파일 기반 설정 로딩, 재시도 / 타임아웃 처리, 암·복호화 포함

#### deferred-deeplink-backend
- Click ID 및 암호화 토큰 기반 Deferred Deep Link 백엔드
- 설치 / 최초 실행 시점 검증 및 위·변조 방지 로직 포함

#### ops-scheduler-batch-jobs
- 이중화 환경을 고려한 Scheduler 기반 배치 처리
- 시간 분산 실행, Lock, Backoff 재시도 구조 반영

#### realtime-caching-gateway
- Redis를 Pub/Sub 브로커를 넘어 **real-time message processing + cache layer**로 확장한 포트폴리오
- PostgreSQL fallback, partial miss recovery, scheduled synchronization 구조 구현

#### spring-boot-jpa-querydsl-api
- JPA / QueryDSL 기반 REST API 설계 예제
- 엔티티 조회 구조 및 API 설계 관점 정리
